Agents forget. Every session. Every tool.

You re-explain the architecture, re-teach preferences, re-discover the same bug — in every agent, every day. Built-in memory files cap out and go stale, and each vendor's memory is their lock-in. memwarden is the memory that follows you across agents, costs fewer tokens to recall, and that you actually own.

🔁

Cross-agent by default

One local daemon, shared over MCP + hooks. Claude, Codex, Cursor, Cline — same brain. Open a tool and it auto-injects the project's memory before you ask.

🗜️

Compressed, not raw

Google's TurboQuant algorithm, implemented in pure TypeScript, shrinks every vector 6–11× with zero recall loss. More memory fits, recall stays fast.

🔐

Verifiable & yours

Every write lands in a SHA-256 hash-chained log you can cryptographically verify. Export your whole brain to a portable bundle. No vendor lock-in.

How it works

Capture → compress → store with integrity → recall the right slice under a token budget.

your agent  ──observe──▶  compress  ──▶  libSQL store + hash-chained oplog

  MiniLM embed  ──▶  TurboQuant 4-bit codes  ──▶  vector index

your agent  ◀──resume / search──  BM25 + vector (RRF), packed under a token budget

Quick start

Two runtime dependencies. No API key. Embeddings run on your machine.

# start the local brain
npm install && npm run build
npm run dev                              # daemon on :3111

# wire it into your agent (from your project dir)
node dist/cli/bin.js connect claude-code --with-hooks

# now switch agents and just ask:
"what were we working on here?"      # it already knows
