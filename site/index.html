Trustworthy, not just stored

The doctor audits whether each memory is still safe to inject: it flags memory that points at deleted files (stale) or has no evidence (unsourced). Auditing trust is the part nobody else leads with.

Verifiable and yours

Every write lands in a SHA-256 hash-chained log you can cryptographically verify. On-device, two dependencies, a portable bundle. No vendor.

Self-healing

Once it is up you never touch it. It revives itself if a tool finds it down, and an OS service restarts it on crash and starts it at login.

Memory you can trust

Recall is only half the job. The other half is knowing the memory is still safe to use. Every memory carries provenance, and memwarden doctor audits it against your live repo. This is the part that makes memwarden its own thing, not another shared-memory layer.

How memory crosses your tools

The honest mechanics. There are exactly three ways memory reaches a tool, and up wires whichever ones each tool supports. We do not pretend any of it is magic.

Door 1 · mechanical

Hooks

Claude Code injects this project's memory before you type, and captures your work as it happens. The agent cannot forget to do it.

guaranteed
Door 2 · standing instruction

MCP + AGENTS.md

For Codex, Cursor, Kiro, Antigravity, OpenCode and OpenClaw, up writes an instruction telling the agent to recall first and save as it goes. It does so on every task. This is how every cross-tool memory layer does these tools, plus /recall when you want to force it.

agent follows it
Door 3 · mechanical

The proxy

Point any model-configurable tool at the memwarden proxy and every turn is recalled and captured, local model or paid, with no agent cooperation at all.

guaranteed

Only hooks and the proxy are mechanically guaranteed. For MCP-only tools, "automatic" means a standing instruction the agent follows, the same mechanism every memory layer uses, backed by /recall on demand.

How it works

Capture, compress on-device, store with integrity, recall the right slice under a token budget. Scoping is by canonical path, so recall never silently misses.

your tool  ──observe──▶  compress  ──▶  libSQL store + hash-chained oplog

  MiniLM embed  ──▶  TurboQuant 4-bit codes  ──▶  vector index

your tool  ◀──resume / recall──  BM25 + vector (RRF), packed under a token budget

Measured, not claimed

Real on-device model (all-MiniLM-L6-v2), 14 paraphrased queries worded differently than the answers. Reproduce with npm run benchmark.

100% is on a small, clean corpus. The point: compression is free (quantized == full-precision) and semantic recall beats lexical. 217 tests, including a cross-tool reliability harness that proves memory captured in one tool is recalled in another.
